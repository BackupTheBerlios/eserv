Каскадирование прокси

Этот модуль предоставляет возможность работы http-прокси Eproxy через другие прокси. В некоторых конфигурациях это единственная возможность обеспечить работу прокси - если прямой доступ в интернет на машине с Eproxy невозможен. Кроме того, иногда работа через другой прокси повышает скорость - например, работа через спутниковый прокси.

Настройка каскадирования производится в файле Eproxy\conf\http-proxy\plugins\hierarchy\OnRequest.rules.txt.

Прокси, через который должен работать Eproxy, будем называть "родительский прокси" (Eproxy, соответственно - дочерний). Можно указывать несколько родительских прокси, и выбор между ними производить на основе любых правил. Родительский прокси указывается в виде 

CascadeVia: proxy.host proxy.port proxy.user proxy.pass.

Если proxy.user proxy.pass не указаны, то авторизация на родительском прокси не производится.

Пример OnRequest.rules.txt: 

\ по умолчанию работа через proxy.isp.ru:
CascadeVia: proxy.ISP.ru 3128

\ если запрашиваются ресурсы из доменов *.com, то работа через 
\ другой прокси, с паролем:
URL =~ *.com/* | CascadeVia: sputnik.dom 3128 sputnik_login passw

\ разрешить прямые соединения, если родительский прокси не работает
DirectConnectionAllowed
