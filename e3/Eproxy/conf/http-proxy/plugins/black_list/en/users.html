If this field is filled, the user name will be checked on coincidence with the mask given in this field also (additionally to a URL/site). The rule works, if both conditions are true. Thus it enables to set rules, used only for a part of users.

If selected authorization method - IP address, the user name will not be compared to a mask, his IP-address will be used instead. For example, it is possible to set a mask "10.1.1.*" in this field.

If the field is not filled, the rule operates for all users.
